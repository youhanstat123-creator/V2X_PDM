# LSTM 모델 구조 쉽게 설명

## 전체 비유: "12초 동안 영상 보고 이상 여부 판단하는 AI"

센서 데이터 12개(=12 타임스텝)를 연속으로 보고 "이 장비 곧 고장날 것 같다/아니다"를 판단하는 구조예요.

---

## 입력 → 출력 흐름

```
입력: 12개 시점 × 9개 센서값
  (응답시간, CPU온도, 오류횟수 ... 12번치)
          ↓
LSTM(64) → LSTM(32) → Dense(32) → Dense(1)
          ↓
출력: 0.0 ~ 1.0  (이상 확률)
  0.1 → 정상일 가능성 높음
  0.9 → 위험할 가능성 높음
```

---

## 각 레이어 한 줄 설명

| 레이어 | 역할 | 비유 |
|---|---|---|
| **LSTM(64)** | 시간 흐름 패턴 학습 | "12초 동안 영상 전체를 기억" |
| **Dropout(0.2)** | 과적합 방지 | "랜덤하게 20% 뉴런 끄기 — 외워서 푸는거 방지" |
| **LSTM(32)** | 핵심 특징만 압축 | "기억한 것 중 중요한 것만 요약" |
| **Dropout(0.2)** | 과적합 방지 | 위와 동일 |
| **Dense(32)** | 비선형 조합 | "요약된 정보로 판단 근거 만들기" |
| **Dense(1, sigmoid)** | 최종 확률 출력 | "0~1 사이 숫자 하나로 결론" |

---

## LSTM이 일반 신경망과 다른 점

```
일반 신경망: 각 시점을 독립적으로 봄
  [t=1] [t=2] [t=3] ... → 각각 따로 판단

LSTM: 이전 시점을 기억하면서 봄
  [t=1] → [t=2] → [t=3] → ... 흐름을 봄
   └─기억─┘  └─기억─┘
```

예지보전에서 중요한 이유: **CPU가 서서히 올라가다가 갑자기 오류 급증** 같은 **시간적 패턴**을 잡아내려면 연속된 흐름을 봐야 하기 때문이에요.

---

## `return_sequences` 차이

```python
LSTM(64, return_sequences=True)
  → 12개 타임스텝 결과 전부 다음 레이어로 전달
  → "12프레임 영상 전체를 다음 레이어에 넘김"

LSTM(32, return_sequences=False)
  → 마지막 타임스텝 결과만 전달
  → "영상 다 보고 나서 최종 요약본 하나만 남김"
```

---

## sigmoid 출력이 임계값(threshold)과 만나는 방식

```
score = 0.73  (모델 출력)
threshold = 0.35  (학습 시 튜닝된 값)

0.73 >= 0.35  → 이상(위험) 판정
0.20 >= 0.35  → 정상 판정
```

임계값이 0.5가 아닌 이유: **위험을 놓치는 게 더 위험**하기 때문에 recall(재현율) 우선으로 낮게 설정해요.
